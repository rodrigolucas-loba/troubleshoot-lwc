Alterações efectuadas
1. Foi adicionado o ficheiro de ruleset PMD custom em config/pmd-ruleset.xml.
2. Foi adicionado o ficheiro de configuração do Code Analyzer em code-analyzer.yml.
3. Foi adicionada a workflow de GitHub Actions em .github/workflows/salesforce-code-analyzer.yml.
4. O ficheiro scripts/run-salesforce-analyzer.ps1 foi actualizado para correr os selectors Recommended e CustomPMD.
5. Os ficheiros docs/code-analyzer-report.csv e docs/code-analyzer-report.md foram regenerados após a activação do ruleset custom.

Medidas tomadas
1. O ruleset PMD custom passou a ficar versionado dentro do repositório, para poder ser usado localmente e em CI.
2. O Code Analyzer passou a carregar o ruleset PMD custom através do ficheiro code-analyzer.yml, em vez de depender de um ficheiro local fora do repositório.
3. A workflow de GitHub Actions passou a chamar o Code Analyzer com a opção --config-file code-analyzer.yml.
4. O quality gate do pull request foi alinhado com o requisito de falhar apenas em findings de severidade Critical.
5. A workflow continua a publicar artefactos do analyzer e comentários no pull request, para manter visibilidade durante a revisão.

Resultado actual da validação
1. O analyzer correu com sucesso usando os selectors Recommended e CustomPMD.
2. A regra custom CamelCaseMethodNaming foi confirmada no CSV gerado.
3. O relatório actual continua a incluir um finding Critical vindo do engine eslint, não relacionado com o ruleset PMD em si.

Notas
